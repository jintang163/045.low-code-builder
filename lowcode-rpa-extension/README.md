# 低代码RPA录制助手 - Chrome扩展

用于录制浏览器操作（点击、输入、抓取数据），自动生成RPA自动化脚本的Chrome扩展。

## 功能特性

- 🎬 **真实录制**：捕获所有浏览器交互操作（点击、输入、下拉选择等）
- 🎯 **智能选择器**：自动生成稳定的CSS选择器（优先使用ID、data-testid等）
- 📥 **数据抓取**：Shift+Ctrl点击元素提取数据
- 📷 **快捷截图**：Ctrl+Shift+E 一键截图
- ⏳ **等待控制**：Ctrl+Shift+W 插入等待步骤
- 💾 **导出脚本**：自动生成标准RPA JSON脚本，可直接导入低代码平台
- 🔄 **暂停/继续**：Esc键快速暂停或继续录制
- 📋 **右键菜单**：快速访问常用录制功能

## 安装方法

### 方法一：开发者模式加载（推荐开发使用）

1. 打开Chrome浏览器，进入 `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本扩展所在目录（`lowcode-rpa-extension/`）
5. 扩展加载成功，会在工具栏显示🤖图标

### 方法二：打包安装（生产环境）

1. 在扩展管理页面点击「打包扩展程序」
2. 选择扩展目录，生成 `.crx` 文件
3. 将 `.crx` 文件拖入Chrome扩展页面安装

## 使用方法

### 基础录制流程

1. 打开要录制操作的网站
2. 点击Chrome工具栏的🤖图标打开扩展面板
3. 点击「开始录制」按钮
4. 在页面上进行正常的操作（点击、输入、下拉选择等）
5. 每一步操作都会被自动记录
6. 录制完成后点击「停止录制」
7. 复制生成的JSON脚本或下载到本地
8. 将脚本粘贴到低代码平台的RPA脚本设计器中

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Shift + Ctrl + 点击` | 提取点击元素的数据 |

| `Ctrl + Shift + W` | 插入等待2秒的步骤 |
| `Esc` | 暂停/继续录制 |

### 右键菜单

在页面任意位置右键，可以快速访问：
- 🎬 开始录制
- ⏹️ 停止录制
- 📥 提取此元素数据
- 📷 截取当前页面
- ⏳ 插入等待步骤

## 录制最佳实践

### 1. 选择器优化

扩展会按以下优先级生成选择器：
1. `#id` - 元素ID（最稳定）
2. `[data-testid="xxx"]` - 测试专用属性
3. `[data-id="xxx"]` - 自定义数据ID
4. `tag[name="xxx"]` - name属性
5. 带class的选择器
6. XPath（最后兜底）

**建议**：开发网页时尽量为关键元素添加 `data-testid` 属性。

### 2. 录制步骤

1. **先登录**：录制前先登录目标系统
2. **从首页开始**：尽量从网站首页开始录制
3. **添加等待**：页面跳转后手动插入等待步骤
4. **使用变量**：对于需要变化的输入值，使用 `${变量名}` 占位符
5. **验证步骤**：关键操作后添加数据提取步骤验证结果

### 3. 常见问题

**Q: 录制时元素选择器不稳定怎么办？**
A: 检查网站是否使用了动态生成的class或id。可以使用开发者工具手动调整选择器。

**Q: 录制的脚本回放失败？**
A: 可能的原因：
- 页面加载速度不同，添加适当的等待步骤
- 元素定位变化，检查选择器是否有效
- 网站有防自动化检测，尝试添加延时

**Q: 如何处理登录弹窗？**
A: 建议录制包含登录的完整流程，或在录制前先登录。

## 生成的脚本格式

```json
{
  "steps": [
    {
      "action": "navigate",
      "url": "https://example.com/login",
      "timeout": 60000
    },
    {
      "action": "input",
      "selector": "#username",
      "value": "${username}",
      "timeout": 30000
    },
    {
      "action": "input",
      "selector": "#password",
      "value": "${password}",
      "timeout": 30000
    },
    {
      "action": "click",
      "selector": "#login-btn",
      "timeout": 30000
    },
    {
      "action": "wait",
      "seconds": 3
    },
    {
      "action": "extract",
      "selector": ".welcome",
      "fieldName": "welcomeText",
      "extractType": "text",
      "timeout": 30000
    }
  ]
}
```

## 导入到低代码平台

1. 在扩展中点击「复制JSON」
2. 进入低代码平台 → RPA自动化 → 新建脚本
3. 填写脚本基本信息后进入设计器
4. 点击「查看代码」按钮
5. 粘贴复制的JSON内容
6. 调整步骤顺序和参数
7. 保存并验证脚本
8. 发布后即可在业务逻辑中作为RPA节点使用

## 权限说明

扩展需要以下权限：

- **activeTab**：访问当前活动标签页
- **storage**：存储录制的脚本历史
- **scripting**：注入录制脚本到页面
- **tabs**：获取页面URL变化
- **webNavigation**：监听页面导航
- **<all_urls>**：在所有网站上录制操作

扩展严格保护用户隐私，所有数据仅存储在本地浏览器中，不会上传到任何服务器。

## 故障排除

### 扩展图标不显示
- 检查是否在扩展管理页面启用了扩展
- 尝试重启Chrome浏览器

### 无法开始录制
- 刷新当前页面后重试
- 检查是否在Chrome网上应用店页面（该页面禁止注入脚本）
- 打开开发者工具查看Console是否有错误

### 录制的步骤为空
- 确认页面上的操作是可交互元素
- 检查Console是否有JavaScript错误
- 尝试刷新页面后重新录制

### 选择器无效
- 手动编辑脚本，调整选择器
- 使用开发者工具重新获取元素选择器
- 尽量使用更稳定的定位方式（如data-testid）
